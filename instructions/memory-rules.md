# Правила памяти и самообучения

> Как агент помнит ученика и становится умнее от раза к разу. CLAUDE.md ссылается сюда.
> Простыми словами: у агента есть память трёх уровней и привычка записывать важное. Благодаря этому он не начинает каждый раз с нуля.

---

## Три уровня памяти

Представь три ящика, куда агент складывает информацию о тебе и твоём контенте.

### Уровень 1 - Постоянная память (`MEMORY.md`)
Что верно ВСЕГДА: твоя ниша, продукт, площадки, важные решения. Лимит - 200 строк, только самое главное.
- **Читать:** всегда при старте сессии.
- **Писать:** появился новый постоянный факт (решил нишу, выбрал продукт) - агент сразу дописывает сюда одной строкой.

### Уровень 2 - Дневник (`memory/ГГГГ-ММ-ДД.md`)
Хроника по дням: что делали, где остановились. Один файл = один день.
- **Читать:** последний дневник при старте (чтобы вспомнить, где закончили).
- **Писать:** в конце сессии - итог дня.
- **Правки ученика** идут в отдельный файл `memory/feedback_<тема>.md` (см. ниже).

### Уровень 3 - Копилка знаний (`knowledge/`)
Базы, которые растут: твоя аудитория, темы, банк идей, продукт, аналитика «что зашло», живой язык аудитории, выводы из опыта.
- **Читать:** под конкретную задачу, точечно. Не тащить всё подряд - лишний файл только мешает.
- **Писать:** когда накопилось что-то полезное на будущее (см. «Самообучение»).

Рядом с памятью при старте читаются ещё два файла (это не память, а «кто есть кто»):
- `SOUL.md` - каким тоном говорит агент и по каким правилам пишется контент.
- `USER.md` - кто ты (ниша, темы, площадки, твой голос).

## Правило дневника (обязательно)

**Каждый новый день работы заводи дневник.** Создай файл `memory/ГГГГ-ММ-ДД.md` (например `2026-07-07.md`). В конце сессии допиши три строки:

1. **Что сделали** - какой результат получили.
2. **Что осталось** - что доделать в следующий раз.
3. **Решения** - что важного решили.

Если файла за сегодня ещё нет - создай его сам на первом рубеже, не жди команды. Дневник - это память «где мы остановились»: в следующий раз откроешь последний и сразу вспомнишь, на чём стоп.

Шаблон дневника:

```markdown
# Дневник ГГГГ-ММ-ДД

## <Главный результат дня одной строкой>

### Что сделали
- ...

### Что осталось
- ...

### Решения
- ...
```

## Самообучение - как агент копит знания от раза к разу

Агент не просто помнит - он учится. После каждой работы важное отправляется в нужный ящик. Это происходит **без спроса** (записывать в свою память - можно всегда).

| Когда | Куда пишем | Что пишем |
|---|---|---|
| Ученик принёс цифры по ролику/посту | `knowledge/what-worked.md` | что зашло + гипотеза почему |
| Ученик поправил агента («не так», «переделай») | `memory/feedback_<тема>.md` | было → как правильно |
| Ученик 2+ раза спросил одно и то же | нужный файл в `knowledge/` | готовый ответ, чтобы не переспрашивать |
| Услышали живую фразу или боль аудитории | `knowledge/audience-voice.md` | дословную цитату |
| Придумали идею, но снимаем не сейчас | `knowledge/idea-bank.md` | тема + крючок |
| Поняли устойчивый вывод из опыта | `knowledge/lessons-learned.md` | контекст → вывод → как меняем подход |

### Шаблон правки ученика (`memory/feedback_<тема>.md`)

```markdown
# Правка: <тема>
Дата: ГГГГ-ММ-ДД
Было: <что агент сделал не так>
Правка ученика: <что человек сказал изменить>
Правильно: <как теперь делаем>
```

Смысл: один раз поправил - агент запомнил навсегда и больше не наступает на те же грабли.

## ⚠️ Важно: две памяти - не путать

У Claude Code есть своя встроенная авто-память - она пишет в служебные папки программы. Это **НЕ наша система**.

Наша память - только эти папки внутри проекта:
- `MEMORY.md`
- `memory/`
- `knowledge/`

Веди записи только сюда. В служебную авто-память ничего писать не надо.
